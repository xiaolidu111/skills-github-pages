回溯算法秒杀所有组合子集排列问题

框架

无论是排列组合还是子集问题,捡来来说无非就是让你从序列nums中以给定规则取若干的元素,主要有以下几种变体

  • 元素无重复不可复选 ,就是说nums中的元素都是位移的,每个元素最多只能用一次,这也是最基本的形式
  • 元素可重复不可复选,就是说nums中的元素可以重复,每个元素最多只能被使用一次
  • 元素无重复可复选,就是说nums中的元素都是位移的,但是每个元素可以被使用若干次

无论形式怎么变化,其本质就是穷举所有的解,这些解呈现出树形结构,所以合理使用回溯算法框架,稍微修改框架就可以把这些问题一网打尽

其实 组合问题和子集问题其实是等价的,置于之前所的三种形式变化,无非就是在这两棵树上剪掉或者增加一些树枝罢了

子集(元素无重复不可复选)

78. 子集

因为子集是不考虑顺序的,[1,2]和[2,1]其实是同一个子集,为了让子集不出现重复,可以使用保证相对顺序来防止不出现重复的子集

代码如下:

var subsets = function(nums) {
    let result = []
    let path = []
    function backTracking(startIndex){
        result.push([...path]) // 加入到result中
        for(let i = startIndex;i < nums.length;i++){
            path.push(nums[i])
            backTracking(i+1) // 从下一位开始保持先后顺序不变
            path.pop()
        }
    }
    backTracking(0)
    return result
};

主需要针对上面求子集的代码,修改一下退出回溯的条件就好了 

var combine = function(n, k) {
    let result = []
    let path = []
    function backTracking(startIndex){
        if(path.length === k){ // 只需要修改一下退出的条件就行
            result.push([...path])
            return
        }
        for(let i = startIndex;i <= n;i++){
            path.push(i)
            backTracking(i+1)
            path.pop()
        }
    }
    backTracking(1)
    return result
};

排列(元素无重不可复选)

上面所讲的组合子集问题,是使用startIndex来保证元素start之后只会出现start+1中的元素,通过固定元素的相对位置保证元素不出现重复的元素

但是排列问题的本质是让穷举元素的位置,nums[i]之后也可以再出现i之前的元素,所以之前的方法是不适用的,需要使用一个used来记录那些元素使用过,那些元素没使用过

子集/组合(元素可重不可复选)

以上讲的题目都是没有重复元素的,但是如果存在重复元素,怎么处理呢

如以下例题

90. 子集 II

如果以1,2,2举例,为了重复区分,可以将其视作1,2,2` 那么这个树可以简化为

可以发现,两条值相同的相邻树枝会产生重复

所以需要对其进行剪枝,也就是一个节点有多条值相同的树枝相邻,则只遍历第一条,剩下的都剪掉,不要去遍历

体现在代码上,就是需要对其进行排序,让相同的元素靠在一起,如果发现后一个和前一个相同 就调过

代码如下

var subsetsWithDup = function(nums) {
    let result = []
    let path = []
    nums.sort((a,b)=>a - b)
    function backTracking(satrtIndex){
        result.push([...path]) // 放入这个path数组
        for(let i = satrtIndex;i < nums.length;i++){ 
            if(i > satrtIndex && nums[i] === nums[i-1]){// 注意这个是要大于其实位置 因为如果大于0的话 会少很多的数字
                continue // 如果这个值大于0 并且这个值和前一个相同 那就直接调过
            }
            // 否则的话
            path.push(nums[i])
            backTracking(i+1)
            path.pop()
        }
    }
    backTracking(0)
    return result
};
40. 组合总和 II 


var combinationSum2 = function(candidates, target) {
    let path = []
    let result = []
    candidates.sort((a,b)=>a - b) // 排序一下
    function backTracking(startIndex,sum){
        if(sum === target){
            result.push([...path])
            return // 如果和是这个 那就加入到结果数组
        }
        if(sum > target){
            return // 如果大于这个target 就直接返回
        }
        for(let i = startIndex;i <candidates.length;i++){
            if(i >startIndex && candidates[i] === candidates[i-1]){
                continue
            }
            sum += candidates[i]
            path.push(candidates[i])
            backTracking(i+1,sum)
            sum -= candidates[i]
            path.pop()
        }
    }
    backTracking(0,0)
    return result
};

排列元素可重不可复选

排列问题如果存在重复值,就比子集.组合问题稍微复杂一点

如下题

47. 全排列 II

对比之前的全排列,只有两个不同

  1. 对这个数组进行了排序
  2. 添加了一句额外的剪枝逻辑

和自己组合问题的逻辑不同,新增了!used[i-1]的逻辑判断

假设输入为 nums = [1,2,2’]，标准的全排列算法会得出如下答案：

显然，这个结果存在重复，比如 [1,2,2’] 和 [1,2’,2] 应该只被算作同一个排列，但被算作了两个不同的排列。

怎么设计剪枝逻辑让这个重复的给去除掉呢

保证相同的元素在排列中的相对位置保持不变

比如说,让2永远在2`的前面,这样智能筛选出三个条件,这也就是正确的答案

这是因为标准的全排列算法之所以会重复,是因为吧相同元素形成的排列序列视为了不同的序列,但是实际上他们是相同的,如果固定相同的元素形成的序列顺序,当然就避免了重复

反应到代码上就是i > 0 && nums\[i\] == nums\[i - 1\] && !used\[i - 1\],如果前面的相邻相等元素没有用过就跳过,这样就避免了重复的排列 

var permuteUnique = function(nums) {
    let result = []
    let path = []
    let used = []
    nums.sort((a,b)=>a-b)
    function backTracking(){
        if(path.length === nums.length){
            result.push([...path])
            return
        }
        for(let i = 0;i < nums.length;i++){
            if(used[i]){
                continue
            }
            if(i > 0 && nums[i] == nums[i-1] && !used[i-1]){
                continue
            }
            used[i] = true
            path.push(nums[i])
            backTracking()
            used[i] = false
            path.pop()
        }
    }
    backTracking()
    return result
};

子集组合(元素无重可复选)

数组无重复元素,每个元素可以被重复使用

例题:

39. 组合总和

首先思考,我们在做这个之前不可复选的时候,是怎么来解决不能复选的,是通过startindex+1来进行的

那么我们只需要将这个+1给去掉就行了

代码如下

var combinationSum = function(candidates, target) {
    let result = []
    let path = []
    function backTracking(startIndex,sum){
        if(sum === target){
            result.push([...path])
            return
        }
        if(sum > target){
            return 
        }
        for(let i = startIndex;i < candidates.length;i++){
            sum += candidates[i]
            path.push(candidates[i])
            backTracking(i,sum)
            sum -= candidates[i]
            path.pop()
        }
    }
    backTracking(0,0)
    return result
};

排列(元素无重可复选)

力扣上没有类似的题目，我们不妨先想一下，nums 数组中的元素无重复且可复选的情况下，会有哪些排列？

比如输入 nums = [1,2,3]，那么这种条件下的全排列共有 3^3 = 27 种：

标准的全排列 用used数组进行剪枝,避免重复使用一个元素,如果允许重复使用的话,直接放飞自我,去除所有的used数组的剪枝逻辑就行了

代码如下

var permuteUnique = function(nums) {
      let result = []
      let path = []
      function backTracking(){
              if(path.length === nums.length){
                      result.push([...path])
                      return
              }
              for(let i = 0;i < nums.length;i++){
                      path.push(nums[i])
                      backTracking()
                      path.pop()
              }
      }
      backTracking()
      return result
};
