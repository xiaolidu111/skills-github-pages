封装骨架屏组件

思路:需要一个组件做,占位使用,这个占位组件有个专业的属于,叫做骨架屏组件

  • 暴露一些属性:高/宽/背景,是否有闪动动画

这是一个公共组件需要全局注册,将来这样的组件建议在vue插件中定义

通常要有一个单独的目录,来存放我们自己定义的组件,一般是叫做library

在这个目录里面单独生成一个组件文件,在文件里面编写效果

  • 组件内部需要可以放置一个盒子,这个盒子的宽高和动画效果可以由外界传入,由外界来控制这个组件的大小和动画

  • 扩展vue原有的功能,通过插件的形式将这个组件挂载到vue上,方便使用

    • vue2中,导出一个对象,有install函数,传入vue的构造函数,vue基础上扩展
    • vue3中,导出一个对象,有install函数,传入的是app应用实例,app基础之上扩展
    • 主要步骤:导入这个组件,然后通过app.component来进行注册

  • 使用

    • 到组件内部使用的时候,监听是否有数据,有数据就显示数据,没有数据的话就显示组件,可以使用v-if和v-show来控制是否显示

复选框组件的封装

思路

  • 实现组件本身选中与不选中的效果
  • 实现组件的v-model 指令
  • 改造成vueuse/core的函数写法

这个复选框是一个通用的组件,可以在这个library文件夹里面去写这个代码

组件实现步骤

  1. 准备两个图标,一个是选中的一个是不选中的,通过一个数据checkd来进行控制图标的显示和隐藏
  2. 给这个组件的最外层的盒子绑定一个点击的时间,用过修改这个checked的状态,后面根据checked 状态修改图标的显示
  3. 对于后面的内容,可以通过默认插槽的方式来传入内容,可以通过$slots.default来判断是否有插槽内容
  4. 组件实现v-model双向数据绑定
    i. v-model有解析成一个语法糖,就是:modelValue和@update:modelValue

ii.这个modelValue的值需要和checked保持一致,所以可以使用watch监听器来完成两者的同步

当数据发生改变的时候,需要通过emit来通知父组件进行相应数据的更新

vue3.0 组合api的写法,用到了一个库@vueuse/core

  • 通过这个库提供的一个api叫做useVModel,这个接受三个参数

    • props
    • 字段名字:modelValue,
    • 方法:emit

  • 当这个被点击的时候 就可以让新的值对原来的值取反就行了,这样会自动的提交给父组件

  • 解释

    • 使用props接受modelValue
    • 使用useVModel来包装props中的modelValue的属性数据
    • 在使用checkd.value的时候就是使用了父组件的数据
    • 在使用checked.value= 数据的时候 内部就会触发emit函数而且就是触发update:modelValue通知父组件改变

还可以通知父组件的change事件

  • 在组件内部通过emit('change')来通知change事件 
<template>
  <div class="xtx-checkbox" @click="changeChecked()">
    <i v-if="checked" class="iconfont icon-checked"></i>
    <i v-else class="iconfont icon-unchecked"></i>
    <span v-if="$slots.default"><slot /></span>
  </div>
</template>
<script>
import { ref } from 'vue'
export default {
  name: 'XtxCheckbox',
  setup () {
    const checked = ref(false)
    const changeChecked = () => {
      checked.value = !checked.value
    }
    return { checked, changeChecked }
  }
}
</script>

实现双向数据绑定

import { ref, watch } from 'vue'
// v-model  ====>  :modelValue  +   @update:modelValue
export default {
  name: 'XtxCheckbox',
  props: {
    modelValue: {
      type: Boolean,
      default: false
    }
  },
  setup (props, { emit }) {
    const checked = ref(false)
    const changeChecked = () => {
      checked.value = !checked.value
      // 使用emit通知父组件数据的改变
      emit('update:modelValue', checked.value)
    }
    // 使用侦听器，得到父组件传递数据，给checked数据
    watch(() => props.modelValue, () => {
      checked.value = props.modelValue
    }, { immediate: true })
    return { checked, changeChecked }
  }
}

@vueuse/core实现

import { useVModel } from '@vueuse/core'
// v-model  ====>  :modelValue  +   @update:modelValue
export default {
  name: 'XtxCheckbox',
  props: {
    modelValue: {
      type: Boolean,
      default: false
    }
  },
  setup (props, { emit }) {
    // 使用useVModel实现双向数据绑定v-model指令
    // 1. 使用props接收modelValue
    // 2. 使用useVModel来包装props中的modelValue属性数据
    // 3. 在使用checked.value就是使用父组件数据
// 4. 在使用checked.value = '数据' 赋值，触发emit('update:modelvalue', '数据')
    const checked = useVModel(props, 'modelValue', emit)
    const changeChecked = () => {
      const newVal = !checked.value
      // 通知父组件
      checked.value = newVal
      // 让组件支持change事件
      emit('change', newVal)
    }
    return { checked, changeChecked }
  }
}

消息提示组件封装

也是一个公共组件,需要放在library目录里面,进行消息提示组件的封装,单独创建一个文件

组件功能分析:

  • 固定顶部显示,有三种类型:成功,错误,警告
  • 显示消息提示需要动画从上滑入且淡出
  • 组件使用方式要封装成工具函数的方式

具体过程

  • 在组件内部封装一个样式对象,对应着不同的样式,键值对的形式,键名是样式名,值是样式对象
  • 外部需要通过props向组件内部传入一个type数据,控制组件的样式,传入的值应该和上面样式对象的值相同
  • 外部也需要传入一个message来控制组件的显示内容
  • 控制组件的显示和隐藏
    • 可以设置一个变量,来控制组件的显示和隐藏
    • 组件渲染完毕的时候,让这个组件显示,可以在onMounted这个声明周期钩子函数中,让这个变量为true,让组件显示
    • 然后给这个加一个动画,结合Transition组件控制显示和隐藏

封装函数调用式

  • 提供一个能够显示xtx-message的函数,这个函数将来可以导入直接使用,也可以挂载到vue的组件实例上

  • 最后的使用方式是直接函数使用或者是this.函数来使用

  • 首先导出一个方法

    • 渲染组件
    • 1 导入消息提示组件
    • 2 将导入的消息提示组件编译为虚拟节点 通过vue提供的createVNode进行转换 两个参数,一个是组件,一个是内容传参
    • 3 准备一个装载消息提示组件的容器
    • 4 将虚拟节点渲染在容器中,render函数渲染虚拟节点,参数1是虚拟节点,参数2是虚拟容器

  • 使用的话,可以在组件里面导入这个函数,就可以完成导入了

    • 可以在install函数中的app.config.globalProperties.$message来完成导入,此时就可以通过this.$message完成使用了,组合api中使用,也可以通过getCurrentInstance()拿到组件的实例完成调用

  • 让这个组件在固定时间消失

    • 设置一个定时器,让他在固定时间后销毁组件,也就是render(null,div)就行了,将null渲染在div中

封装成vue实例函数调用

// 实现使用函数调用xtx-message组件的逻辑
import { createVNode, render } from 'vue'
import XtxMessage from './xtx-message.vue'

// 准备dom容器
const div = document.createElement('div')
div.setAttribute('class', 'xtx-message-container')
document.body.appendChild(div)
// 定时器标识
let timer = null

export default ({ type, text }) => {
  // 实现：根据xtx-message.vue渲染消息提示
  // 1. 导入组件
  // 2. 根据组件创建虚拟节点
  const vnode = createVNode(XtxMessage, { type, text })
  // 3. 准备一个DOM容器
  // 4. 把虚拟节点渲染DOM容器中
  render(vnode, div)
  // 5. 开启定时，移出DOM容器内容
  clearTimeout(timer)
  timer = setTimeout(() => {
    render(null, div)
  }, 3000)
}

分页组件

大致步骤

  • 分页基础布局
  • 分页内部逻辑，完成切换效果
  • 接收外部数据，提供分页事件

具体过程

一些问题:

  • 上一页什么时候禁用 ,当当前页是第一页的时候禁用
  • 前面的…什么时候显示,当显示的页面第一个按钮大于1,就显示出这个…
  • 中间的按钮怎么显示呢,根据起始页码和结束页码得到一个数组,来控制中间页码的显示
  • 最后一个…怎么显示,需要判断最后一页要小于这个最大页,就让他显示…
  • 总结,需要按钮个数,当前页码,总条数,单页条数,总页数,起始页码,结束页码,按钮数组才可以完成渲染

创建一个单独的分页文件

  • 总页数:数据条数/单页显示数据,向上取整

  • 首先定义一下,计算起始页码和结束页码的偏移数量,就是显示多少页/2,向下取整

  • 起始页码: 理想情况下:当前页-便宜数量

    • 特殊情况:如果起始页码小于1 让start=1,只能等于1
    • 特殊情况: 如果结束页码 > 总页数 那么只能让结束页码等于总页数

  • 结束页码: 理想情况下:起始页码+总条数

    • 特殊情况: 结束页码 > 总页数 那么这个end就应该等于总页数
    • 特殊情况: 开始页码 应该判断是不是<1 如果小于1 那么只能等于1

  • 根据起始页码和结束页码,得到按钮数组,从其实到结束,返回给页面,渲染数据

  • 按钮控制

    • 上一页:当当前的页面大于1的时候才可以点,小于等于1的时候不能点
    • 前面的…: 当起始页 > 1 才显示…
    • 后面的…: 当结束页 < 最大页 才显示
    • 下一页: 当前页 < 最大页的时候才可以点击,大于或等于最大页的时候 不能点击

  • 点击事件,直接修改当前页的页码值就行了,上一页–,下一页++

支持外部的传入的功能

  • 外部传入数据,就需要定义props

    • total
    • pagesize
    • currentPage

  • 监听props 数据的变化,更新组件内的数据

    • 改变我们前面定义的total
    • 改变前面定义的pagesize
    • 改变前面定义的当前页

  • 在使用的时候传入相关的数据就可以了

点击影响数据切换

  • 在点击的时候,将分页的页码数据传递给父组件,通过emit方法
  • 可以设置这个名字是current-change,传递出这个页码
  • 父组件使用的时候,接受参数,修改父组件中的页码数据,根据新页码发请求
  • 可以在父组件监听这个页码的改变,一改变就重新发请求
